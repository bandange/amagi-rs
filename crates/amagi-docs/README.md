# amagi-docs

This workspace crate hosts the Amagi-rs documentation site.

It is intentionally marked `publish = false` because the documentation site is
not part of the public Rust API surface.

The site is built with Dioxus and embeds Markdown from the repository-level
`docs/` directory at compile time.

## Run Locally

Install the Dioxus CLI with the official installer, which uses prebuilt
artifacts instead of compiling the CLI locally:

```bash
curl -fsSL https://dioxus.dev/install.sh | bash
```

Serve the docs site from the workspace root:

```bash
dx serve --package amagi-docs
```

The dev server watches:

- `crates/amagi-docs/src`
- `crates/amagi-docs/assets`
- `docs`

## Build

Build the static web bundle:

```bash
dx build --package amagi-docs --release
```

The output is written to `crates/amagi-docs/dist`.

## Content

Source Markdown is kept under `docs/`:

- English: `docs/en`
- Chinese: `docs/中文`

The site uses Dioxus Router for shareable documentation URLs:

- `/zh`
- `/zh/cli`
- `/en`
- `/en/rust-api`

The sidebar is generated from the embedded document index and navigates through
internal Dioxus routes instead of linking directly to Markdown files.

The sidebar search runs entirely client-side over the embedded page index and
Markdown body content for the active language.

The root `DISCLAIMER.md` and `DISCLAIMER.zh-CN.md` files are rendered as
localized documentation pages and linked from the site footer.

Markdown links that point at another local `.md` file are rewritten during
rendering so links inside document content also resolve to Dioxus routes such as
`/zh/cli` or `/en/installation`.

Code blocks are highlighted in the browser with local Highlight.js assets and
enhanced with copy buttons by a small DOM observer after client-side route
changes.

## Theme

The documentation UI uses Material Design 3 color tokens generated by
`material-color-utilities-rs` from a fixed Amagi-rs source color. The dependency is
pinned to the upstream Git commit used during integration because the crate name
on crates.io currently points at a different repository.

Rust generates `--md-sys-color-*` variables for light and dark schemes, while
`assets/main.css` maps those tokens into the site-level semantic variables such
as `--bg`, `--surface`, `--text`, and `--accent`.
