# Amagi-rs 测试分层

英文版：[../en/testing.md](../en/testing.md)

## 公开测试：`crates/amagi-testkit`

`amagi-testkit` 是提交到仓库的测试专属 crate。它是 workspace member，`publish = false`，默认安全，可在 CI 中运行。

适合放在这里的测试：

- public API 兼容测试
- feature matrix 测试
- 服务接口路由和 JSON 字段契约测试
- CLI help/version smoke
- 不依赖 cookie、不访问真实上游、不包含真实响应数据的测试

常用命令：

```bash
cargo test -p amagi-testkit
cargo test -p amagi-testkit --no-default-features --features catalog --tests
cargo test -p amagi-testkit --no-default-features --features adapters --tests
cargo test -p amagi-testkit --no-default-features --features server --tests
```

## 私有测试：`crates/amagi-private-tests`

`amagi-private-tests` 是本地私有 crate。它被 `.gitignore` 忽略，不提交，不加入 workspace，只用于真实 cookie、真实上游和私有 fixture。

本地运行：

```bash
cargo test --manifest-path crates/amagi-private-tests/Cargo.toml
cargo test --manifest-path crates/amagi-private-tests/Cargo.toml -- --ignored
```

安全规则：

- 真实数据测试必须默认 `#[ignore]`。
- cookie/token 只从环境变量或本地 `.env.test.local` 读取。
- `.env.test.local`、`private-fixtures/`、HAR 文件和原始响应不提交。
- 不打印完整 response，不写入包含真实数据的 snapshot。
- 断言结构和关键行为，不把真实 payload 作为测试基线。

本地 `.env.test.local` 示例：

```dotenv
AMAGI_DOUYIN_COOKIE=
AMAGI_PRIVATE_DOUYIN_AWEME_ID=
```

私有 crate 放在仓库目录下只是为了复用本地 path dependency。它的 `Cargo.toml` 内包含空 `[workspace]` 表，避免被根 workspace 自动吸收。
